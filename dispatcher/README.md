# Perfect Moment Dispatcher — סוכן מחבר

הסוכן שמחבר בין הלקוח (סרטון בוואטסאפ) לבין ה-worker (הפייפליין המקומי),
ומחזיר ללקוח את התמונות. עלות: ‎$0/חודש — הכל מקומי.

## Runbook (איך משתמשים)

1. לקוח שולח סרטון בוואטסאפ (972547676000).
2. שומרים את הקובץ לתיקייה `dispatcher/inbox/`.
3. מריצים: `/perfect-moment-run` (או אומרים ל-Claude Code בתיקייה הזו: "process the inbox").
4. הסוכן: מזהה את הסרטון → מריץ את הפייפליין → **בודק את התמונות ויזואלית** → שם את
   התוצאות ב-`done/<job>/` כולל `reply.txt` בעברית.
5. פותחים את `reply.txt`, מדביקים בוואטסאפ, מצרפים את התמונות. זהו.

## מבנה

```
dispatcher/
├── CLAUDE.md        # המוח של הסוכן — ה-workflow המלא
├── inbox/           # לכאן נכנסים סרטונים חדשים
├── processing/      # עבודה בתהליך (זמני)
├── done/<job>/      # תמונות מאושרות + reply.txt + manifest
├── failed/<job>/    # שגיאות / איכות נמוכה — לבדיקה ידנית
├── jobs.csv         # לוג כל העבודות
└── tools/
    ├── scan_inbox.py     # זיהוי + ולידציה של קבצים חדשים
    ├── run_pipeline.py   # הרצת python -m perfectmoment extract
    └── finalize_job.py   # אריזה: --approve (למסירה) או --review (לבדיקה)
```

## מה הסוכן מוסיף מעבר לפייפליין?

הפייפליין מדרג לפי מספרים (חדות, עיניים, חיוך, אסתטיקה). הסוכן **מסתכל בפועל**
על התמונות שנבחרו ופוסל פריים עם עיניים עצומות / טשטוש / חיתוך מוזר לפני
שהוא מגיע ללקוח. פחות מ-3 תמונות טובות → העבודה עוברת ל-failed/ לבדיקה שלך.

## מקרי קצה

| מקרה | מה קורה |
|---|---|
| קובץ שאינו וידאו (txt, jpg…) | עובר ל-`failed/` עם reason.txt |
| וידאו פגום / הפייפליין נכשל | `failed/<job>/error.log` עם ה-stderr |
| `quality_bar_met: false` | `failed/<job>/` + שורת needs_review בלוג |
| שם קובץ כפול | סיומת timestamp אוטומטית |
